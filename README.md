# Дашборд пресейла ОГВ

## Что это

Веб-дашборд для снятия метрик пресейла ОГВ из ежемесячной выгрузки «Статистика по выполненным работам».

Заменяет ручной перенос данных в «Значения показателей»: загружаешь большой xlsx — дашборд автоматически считает **5 показателей** (Поступило, Проработано, Ср. трудоёмкость, Ср. длительность, Ср. на контроле) в разрезах **Услуга / Продукт / Масштаб / Инициатор / Команда**, с фильтрами, drill-down до конкретных запросов и выгрузкой привычного Excel-отчёта.

---

## Стек

- **Python 3.10+**, FastAPI, Uvicorn, openpyxl, SQLite
- **Фронтенд**: Vanilla JS + Chart.js (локально, без CDN)
- **Тесты**: pytest + Playwright

---

## Запуск

**Через скрипт (рекомендуется для Windows):** двойной клик на `launch.bat` или из терминала:

```bat
launch.bat
```

**Напрямую:**

```powershell
.venv\Scripts\python.exe -m uvicorn src.main:app --host 0.0.0.0 --port 8090 --reload
```

Открыть в браузере: **http://localhost:8090/**

---

## Запуск в Docker

Образ работает офлайн (Chart.js вшит локально, без CDN). База SQLite хранится в томе `./data` и переживает пересоздание контейнера.

```bash
docker compose up --build -d
```

Открыть: **http://localhost:8090/**

Остановить:

```bash
docker compose down
```

Без compose:

```bash
docker build -t presale-dashboard .
docker run -d -p 8090:8090 -v "$(pwd)/data:/app/data" presale-dashboard
```

Путь к БД внутри контейнера задаётся переменной `PRESALE_DB` (по умолчанию `/app/data/presale.db`).

---

## Использование

1. Нажать **«Загрузить файл»** и выбрать xlsx-выгрузку «Статистика по выполненным работам».
2. После разбора файла (~1.5–2 мин) дашборд показывает показатели за все месяцы.
3. Переключать вкладки показателей и разрезы (Услуга / Продукт / Масштаб / Инициатор / Команда).
4. Ставить фильтры по услуге, продукту, масштабу, инициатору, команде.
5. Кликнуть по ячейке таблицы — откроется список конкретных запросов (drill-down).
6. **«Выгрузить Excel»** — скачать готовый отчёт со всеми показателями.

## Показатели и правила расчёта

Группировка по **месяцу начала статуса**; дедупликация по ИД запроса.
Подробная спецификация: `docs/superpowers/specs/2026-06-05-presale-dashboard-design.md`.

| Показатель | Что считается |
|---|---|
| **Поступило** | Количество уникальных запросов со статусом «Инициализация», сгруппированных по месяцу инициализации |
| **Проработано** | Количество уникальных запросов со статусом «Принято», сгруппированных по месяцу принятия |
| **Ср. трудоёмкость** | Среднее значение «Отработано часов» по принятым запросам |
| **Ср. длительность** | Среднее суммарной «Длительности проработки запроса (раб. дн.)» по принятым запросам |
| **Ср. на контроле** | Среднее длительности статуса «На контроле» по принятым запросам |

---

## Валидация

Проект содержит golden-тест, который сверяет расчёт с эталонным файлом «Значения показателей (v1)» по всем разрезам:

- счётчики (Поступило, Проработано) совпадают **точно**;
- трудоёмкость — с допуском на 1 переклассифицированный запрос между снимками данных.

---

## Известные ограничения

- Исходный xlsx крупный (десятки МБ, много сводных листов) — разбор одной загрузки занимает **~1.5–2 мин**. Это разовая ежемесячная операция.
- Доступ общий, **без авторизации** (внутренний инструмент).
- Разрез **«Команда»** — новый, в эталонном файле v1 его нет.
